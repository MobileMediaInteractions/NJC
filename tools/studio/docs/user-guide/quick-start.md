# Quick start

1. Run `pnpm studio:start` from the repository root.
2. Choose a folder. The Studio explains detected project types without running code.
3. Open a `.pani` file. Compiler diagnostics update as source changes.
4. Use Code, Visual and State to work on the same canonical document.
5. Select a component in the canvas or virtual device and edit its property in the Inspector. Undo with the toolbar or standard shortcut.
6. Select a timeline, run it, seek the playhead, or open a keyframe to change its time, value and easing.
7. In State, click a transition to inject its event and inspect whether it was accepted from the active state.
8. Toggle device profile, orientation, theme or reduced motion in the toolbar.
9. Open Package, Performance, License Lab or Devices for compiled metadata and diagnostics.
10. Review and trust the workspace before running a displayed task. The exact command and directory are shown in Output.

The virtual phone is an in-process runtime preview, not an operating-system emulator. Real Apple and Android adapters require their vendor SDKs and are not enabled in this milestone.
